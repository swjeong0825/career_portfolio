Things I've shipped, end to end.

Each project below is a working live site. I usually start from a question about architecture — how to keep an LLM useful without putting it in the write path, or how to make structured generation deterministic enough to build on — and let the product fall out of that.

Live · tlmb.swjapps.com

Tennis League Management Bot

A chat-first tennis league app split across three repositories: a domain backend (FastAPI + PostgreSQL) modeled with DDD, an LLM intent service that interprets natural-language messages, and a vanilla static frontend. Writes never go through the LLM — the user always confirms a pre-filled form before mutation.

  • FastAPI
  • PostgreSQL
  • SQLAlchemy
  • Alembic

Auto Bookclub AI

Search a book, get a 6-turn debate between two AI personas with opposing reading philosophies. A two-stage LLM pipeline (persona generation, then sequential per-turn generation) with strict JSON Schema validation keeps the output reliable enough for an interactive continuation flow.
