Auto Bookclub AI

A full-stack AI book-discussion platform that auto-generates structured debates between two LLM-created personas with opposing reading philosophies. Search a book, watch them debate, then inject your own prompts to extend the conversation indefinitely.

auto-bookclub-ai.vercel.app GitHub

Overview

Multi-language search, contrasting personas, deterministic output.

The product is small but tightly designed. Search a book in English (Open Library) or Korean (Google Books) — the language is detected automatically. The system then generates two AI personas with deliberately contrasting reading philosophies for that specific book, and produces a sequential 6-turn debate transcript. Users can inject new prompts mid-discussion to extend it indefinitely, and copy the full transcript out.

The whole UX hinges on output reliability. If even one turn fails to validate, the sequential continuation breaks. So the interesting design work was making LLM output deterministic enough to depend on.

Two-stage LLM pipeline

Personas first, then per-turn debate — both schema-validated.

Splitting persona generation from debate generation makes the system both cheaper and more steerable. Personas are generated once per book and reused across every turn; turn generation only needs the personas, the book, and the running transcript.

[Stage 1] Persona generation Book metadata | v OpenAI Responses API ---- strict JSON Schema (personas) | v { personaA, personaB } (cached for the session) [Stage 2] Sequential debate generation { book, personas, prior turns } | v OpenAI Responses API ---- strict JSON Schema (turn) | v turn_i -> appended to transcript | v (loop until 6 turns, or user injects a new prompt)
Why strict schema mode: The OpenAI Responses API enforces JSON Schema at the token level. This is what makes the sequential continuation flow possible — every turn is guaranteed to be structurally parseable, so the orchestrator can append it and immediately request the next one without defensive parsing or retries.
